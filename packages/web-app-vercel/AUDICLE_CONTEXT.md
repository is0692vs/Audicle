# Audicle UIコンテクスト

## アプリコンセプト

AudicleはWeb記事を高品質TTSで読み上げるアプリです．
- ターゲット: IT技術系記事（Qiita, Zenn等）
- ユースケース: 「ながら聴き」での情報収集
- UXコンセプト: 音楽プレイヤーのような直感的な操作性

## 主要機能

1. **記事一覧**
   - 保存済み記事のリスト表示
   - プレイリスト管理

2. **記事読み上げ**
   - 段落単位での音声再生
   - 再生中箇所のハイライト
   - 自動スクロール追従

3. **プレイリスト機能**
   - 複数プレイリストの作成・管理
   - 記事をプレイリストに追加・削除

## 現在のUIの状態（スクリーンショット確認済み）

### 既に実装されている機能
- **プレイリスト管理**: カード形式で表示，記事数と日付表示あり
- **プレイリスト追加モーダル**: チェックボックス形式で複数プレイリストに同時追加可能
- **新規プレイリスト作成**: 名前と説明を入力するモーダルフォーム
- **ダークモード**: 既に実装済み（ネイビー系の配色）

### 改善が必要なポイント

#### 全体レイアウト
- **サイドバーナビゲーション未実装**: 現在は各ページへの遷移が分かりにくい
- **固定再生バー未実装**: 再生コントロールがページごとに分散
- **統一感のあるレイアウト**: Spotify風の3カラムレイアウト（サイドバー + メイン + 固定プレイヤー）が必要

#### トップページ
- 記事リストが垂直方向にリスト表示のみ
- グリッドレイアウトやカード形式の採用余地
- URL入力欄が目立たない

#### 記事リーダーページ（スクリーンショット確認済み）
- **既存実装**: 段落単位のハイライト表示，再生/一時停止ボタン，プレイリスト追加ボタン
- **改善点**:
  - 再生コントロールが本文と混在している
  - 固定プレイヤーバーがないため，スクロール時にコントロールが見えなくなる
  - 現在再生中の記事情報（タイトル，URL）の表示が弱い
  - プログレスバーの視認性向上
  - 読み上げ速度などの追加コントロール

#### ローディング状態のUI
- **現状**: 「読み込み中...」のシンプルなテキスト表示
- **改善要望**: 
  - モダンなスケルトンローダー
  - スピナーアニメーション
  - プログレッシブローディング
  - ブランドカラーを使ったローディングアニメーション

#### プレイリストページ
- プレイリストカードは既に実装済みだが，より視覚的な改善余地あり
- サムネイルやアイコンの追加
- ホバー時のインタラクション

## 理想的なUI/UX

### デザインコンセプト
- **Spotify風の音楽プレイヤーUI**
- ダークモードベース
- カードベースのレイアウト
- アイコンを使った直感的な操作

### 主要要素
1. **サイドバーナビゲーション**
   - ホーム
   - プレイリスト一覧
   - 設定

2. **固定再生バー（下部）**
   - 現在再生中の記事情報
   - 再生/一時停止ボタン
   - プログレスバー

3. **カードベースの記事リスト**
   - サムネイル（オプション）
   - タイトル
   - 読み上げ時間（推定）
   - プレイリスト追加ボタン
